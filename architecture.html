📊 System Architecture

How DocketIQ processes public comments using AI agents

System Overview

DocketIQ is a multi-agent AI system that analyzes public comments on federal regulations. The system follows a three-step workflow:

Step 1
Docket Selection
User chooses a regulatory docket
→
Step 2
Data Fetching
Retrieve documents & comments from Regulations.gov
→
Step 3
AI Analysis
16 agents process and analyze data
→
Output
Insights Dashboard
Topics, sentiment, campaigns, recommendations

Step 1: Docket Selection

User selects a docket ID to begin analysis

📥 Input

  • User selection from sample dockets
  • Custom docket ID entry
  • Data type indicators (real vs. simulated)

⚙️ Processing

  • Display docket options with badges
  • Validate docket ID format
  • Determine data source
  • Prepare API parameters

📤 Output

  • Selected docket ID
  • Data type flag
  • Transition to Step 2

Step 2: Data Fetching & Snapshot

Retrieve docket metadata, documents, and public comments

Agents Used: docket-fetcher document-fetcher comment-fetcher attachment-processor

📥 Input

  • Docket ID from Step 1
  • Regulations.gov API endpoint
  • API key (if configured)

⚙️ Processing

  • Fetch docket metadata (title, agency, dates)
  • Retrieve document list with attachments
  • Pull public comments (up to 250)
  • Parse attachment metadata
  • Validate and sanitize data
  • Handle zero-comment dockets

📤 Output

  • Docket metadata object
  • Document array (max 10 displayed)
  • Comment array with attachments
  • Visual snapshot UI

Step 3: AI-Powered Analysis

Multi-agent processing pipeline analyzes comments

Agents Used: comment-normalizer duplicate-detector topic-classifier argument-extractor entity-recognizer sentiment-scorer authority-scorer summary-generator timeline-analyzer fraud-detector plain-language-translator

📥 Input

  • Comment array from Step 2
  • Docket metadata
  • Topic registry
  • Configuration settings

⚙️ Processing

  • Normalize & clean comment text
  • Detect duplicate/campaign comments
  • Classify comments by topic
  • Extract arguments and evidence
  • Recognize stakeholder entities
  • Score sentiment & authority
  • Analyze submission timeline
  • Detect fraudulent patterns
  • Generate topic summaries
  • Build knowledge graphs
  • Create recommendations

📤 Output

  • Topic clusters with evidence
  • Sentiment distribution
  • Campaign groups & duplicates
  • Stakeholder breakdown
  • Timeline with surge detection
  • Fraud alerts
  • Knowledge graphs
  • Analyst recommendations
  • ROI estimates

Technical Stack

🎨 Frontend

  • Vanilla JavaScript (ES6 modules)
  • HTML5 + CSS3
  • No framework dependencies
  • Real-time reactive rendering

🔌 APIs

  • Regulations.gov API v4
  • OpenAI GPT-4 (optional, for translation)
  • RESTful architecture
  • API key management via config.local.mjs

🧠 AI/ML Techniques

  • MinHash for duplicate detection
  • Jaccard similarity scoring
  • TF-IDF for topic classification
  • Pattern matching for entities
  • Statistical time-series analysis
  • LLM integration (GPT-4)

📊 Data Processing

  • Multi-stage pipeline architecture
  • Defensive programming with validation
  • Null-safe data handling
  • Comprehensive audit logging
  • Performance optimization

Transparency & Human Oversight

DocketIQ is designed with transparency and human oversight as core principles:

📋 Audit Trail

  • Every agent action logged
  • API calls tracked
  • Data transformations recorded
  • Exportable JSON logs
  • Three detail levels (Summary, Details, Full)

🤖 Agent Transparency

  • Full system prompts visible
  • Explicit capability lists
  • Clear limitation statements
  • Implementation details exposed
  • Call graphs showing agent interactions

👤 Human Oversight

  • AI assists, humans decide
  • Source attribution for all insights
  • Clickable evidence trails
  • Manual verification encouraged
  • Recommendations, not directives

⚠️ Limitations Disclosure

  • Each agent lists what it CANNOT do
  • No black-box AI claims
  • Accuracy expectations set
  • Edge cases acknowledged
  • Continuous improvement tracked
